=== THE JSON EDITING RULE (most common way to wreck this job) ===
NEVER round-trip a whole data file through json.load -> json.dump.
curriculum.json holds ~581 problem objects; a round-trip reformats
indentation, key order and unicode escaping, producing a thousand-line diff
that hides your real change. Instead:
- Use surgical string edits for single-field changes (e.g. one problem's
  "difficulty": "Hard" -> "Medium").
- For genuinely list-shaped changes (reordering stage_order or skill_order),
  hand-write the new list, or use a script that only PRINTS the list and
  paste the result in.
- After ANY data-file edit: `git diff --stat` must show a change
  proportional to the edit (the stage_order reorder is ~a dozen lines, not
  thousands). If it is huge, `git checkout -- <file>` and redo it
  surgically. Then confirm `python3 -m json.tool <file> > /dev/null` parses.
