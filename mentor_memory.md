
# Mentor Memory

## Student Profile

Learning style:
- Socratic.
- One question at a time.
- Derives algorithms instead of memorizing them.
- Responds best to proof-oriented discussion.
- Prefers implementation only after reasoning is complete.

Avoid:
- Revealing algorithm names too early.
- Writing code before invariant discovery.
- Long explanations after successful derivation.

Preferred Session Flow:
1. Present problem.
2. Student restates.
3. Examples.
4. Brute force.
5. Identify repeated work.
6. Discover invariant.
7. Prove correctness.
8. Student designs algorithm.
9. Student writes code.
10. Mentor reviews only.

Hint Policy:
- The single canonical hint ladder lives in `mentor/mentor_protocol.md`; this
  file keeps no separate ladder.

Review Policy:
- Review correctness.
- Review invariant preservation.
- Review interview quality.
- Suggest at most one improvement.
- Move on immediately if solution is correct.
